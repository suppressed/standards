# Security Policy

## Reporting security issues

Please report suspected security vulnerabilities, privacy-impacting bugs, exposed credentials, misconfigurations, or abuse risks by email:

```text
priv@suppressed.com
```

Do not open public GitHub issues for sensitive security reports.

## Please include

When reporting an issue, include as much of the following as possible:

- affected URL, document, asset, or workflow;
- description of the issue;
- reproduction steps;
- observed and expected behavior;
- screenshots, logs, or request details where safe to share;
- assessed impact;
- whether any personal information, credentials, or private data may be involved.

## Responsible disclosure

Please do not:

- access, modify, delete, or exfiltrate data that is not yours;
- degrade, interrupt, or bypass SUPPRESSED systems;
- perform social engineering;
- publish details before SUPPRESSED has had a reasonable opportunity to investigate and remediate.

## Scope

This repository contains public standards materials and supporting assets. Security reports may still be relevant where they involve:

- exposed secrets;
- unsafe links;
- malicious files;
- workflow or repository configuration risks;
- documents or assets that could mislead users or create privacy or security exposure.

## Response

SUPPRESSED will review credible reports and respond where appropriate. Response timing may vary according to severity, operational impact, and platform maturity.
